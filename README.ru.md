**Job Hunter AI** — это автономное десктопное приложение, которое анализирует вакансии, безжалостно отсеивает мусор (скам, MLM, рабство 60 ч/нед) и пишет точечные сопроводительные письма. Всё запускается одной горячей клавишей, в любом браузере, без какого-либо дополнительного ПО.

**В версии 3.0.0 мы навсегда избавились от расширения для Chrome.** Нажмите одну клавишу. Остальное — автоматически.

## ⚙️ Быстрый старт

> Никакого расширения для Chrome. Никакого Flask-сервера. Никаких конфликтов портов. Просто запустите приложение.

**1.** Скачайте последний релиз и запустите **Job Hunter AI.exe** (Windows) или `job-hunter-ai` (Linux).

**2.** Откройте Настройки → введите API-ключ вашего ИИ-провайдера.

**3.** Откройте любую вакансию в любом браузере.

**4.** Нажмите **Ctrl+Shift+J** — приложение само выделит весь текст, скопирует его и отправит в ИИ-конвейер.

**5.** Через ~15 секунд появится всплывающее уведомление: вакансия принята с готовым письмом или отклонена с указанием причины.

Приложение живёт в **системном трее** и работает в фоне — не мешая вам.

## 🚀 История обновлений

<details open>
<summary><b>🟢 v3.0.0 — Автономное приложение. Расширение больше не нужно. (Текущая версия)</b></summary>

<br>

> **В этом релизе расширение для Chrome удалено навсегда.**
> В ранних версиях расширение было удобным костылём, но создавало реальные неудобства: ручная установка, разрешения, проблемы совместимости с Manifest V3, зависимость от конкретного браузера, отдельная кодовая база. В v3.0.0 весь конвейер ввода данных заменён глобальной горячей клавишей, которая работает в любом браузере, на любой ОС, без какого-либо дополнительного ПО.

**Архитектура и новые возможности**

* **[Архитектура]** Удалены расширение Chrome и локальный Flask-сервер. Новый путь ввода: `pynput.GlobalHotKeys` → симуляция `Ctrl+A / Ctrl+C` → `pyperclip.paste()` → `queue.put()`. Никаких открытых портов, никакого браузерного кода.
* **[Автоматизация]** Новый `BrowserCaptureEngine` (`jh_automation.py`): глобальный слушатель горячих клавиш в потоке-демоне, настраиваемая строка hotkey, мягкий откат `AUTOMATION_AVAILABLE` при отсутствии pynput/pyperclip.
* **[Автоматизация]** Раскладко-независимые VK-коды: Windows — Win32 Virtual Key (VK_A=65, VK_C=67, VK_L=76), Linux/X11 — аппаратные коды (A=38, C=54, L=46), macOS — константы kVK. Горячие клавиши срабатывают независимо от активной раскладки.
* **[Платформа]** Официальная поддержка Linux (X11). Добавлен `build_linux.py` с Linux-специфичными hidden imports для X11-бэкендов pynput (`pynput._util.xorg`, `pynput.keyboard._xorg`) и GTK-бэкенда pystray.
* **[Платформа]** Wayland zero-trust guard: `enforce_linux_subsystem_guard()` проверяет `XDG_SESSION_TYPE` и `WAYLAND_DISPLAY`. Бросает `PlatformSecurityException`; перехватывается в `BrowserCaptureEngine.start()` — приложение продолжает работу без макро-функции и выводит чёткое сообщение о решении проблемы (переключитесь на GNOME on Xorg).
* **[Трей]** Иконка в системном трее через `pystray` — приложение работает в фоне, меню по правой кнопке: показать окно / вкл/выкл / выйти.
* **[AI]** Новый конвейер скоринга: `extract_relevant_context()` заменяет `preprocess_vacancy_text()`. Семь шагов: нормализация пробелов → удаление навигационного шума (≤2 слова И ≤25 символов без совпадения с ключевыми словами) → разбивка по `"\n\n"` → скоринг по `_VACANCY_KW_RE` (попадания + `min(len / 600, 2.0)`) → жадный выбор по убыванию → Правило нарратива (восстановление порядка документа) → `pack_paragraphs_to_budget()`. Этапы 1 (12 000 символов) и 2 (8 000 символов) вызывают конвейер независимо — никакого смещения контекста.
* **[AI]** `pack_paragraphs_to_budget()`: строгая упаковка с учётом делимитеров. Жёсткий инвариант `len(result) ≤ max_chars` соблюдается на каждом шаге, включая учёт `len(delimiter)` начиная со второго абзаца. Слишком большие абзацы пропускаются; цикл продолжает собирать меньшие.
* **[Хранилище]** `_write_json_atomic()`: Write-Copy-Replace с `fsync`. `tempfile.mkstemp(dir=тот_же_раздел)` → `json.dump` → `f.flush()` → `os.fsync()` → `os.replace()`. Живой файл никогда не открывается с `O_TRUNC`. Все три пути записи (`_save_file`, `_modify_file`, `_migrate_strip_description`) теперь делегируют этой функции.
* **[Уведомления]** Потокобезопасный Toast: `_notification_lock` защищает мутации `_toast_ref`. `_fade_out_instance` захватывает `toast` напрямую (не глобальный ref) — параллельные уведомления не могут нарушить анимацию угасания друг друга. Обнуляет глобальный ref только если `_toast_ref[0] is toast` (проверка идентичности, не равенства).
* **[Сборка]** `build_exe.py`: исправлен путь к `logo.png` (теперь `assets/logo.png`). `installer.iss`: удалены все блоки, связанные с расширением (`[Files]`, `[Icons]`, `[Run]`, `[CustomMessages]`). `build_linux.py`: новый файл, зеркалит `build_exe.py` с Linux-специфичными hidden imports, `--noconsole`, `os.chmod(0o755)` и баннером системных зависимостей.

**Исправления ошибок, перешедших из 2.0.2**

* **[Фикс]** Тема уведомлений не применялась при запуске: `jh_notifications.apply_theme()` вызывался до импорта модуля — тихий `NameError`, поглощаемый блоком `except Exception`, означал, что цвета Toast всегда оставались захардкоженными, игнорируя выбранную тему.
* **[Фикс]** Молчаливое повреждение конфига Gemini: стартовая миграция переименовывала `"gemini-3.0-pro"` → `"gemini-3.1-pro"`, но `"gemini-3.1-pro"` не существует в `ALL_PROVIDERS_MODELS`. Пользователь с сохранённой этой моделью терял её из выпадающего списка. Ошибочная ветка миграции удалена — `"gemini-3.0-pro"` остаётся валидным именем.
* **[Фикс]** Предупреждение о медленной модели не показывало порог: `tr("warn_text", min_tps=12)` передавал именованный аргумент, которого не было в строке — значение тихо отбрасывалось блоком `except (KeyError, ValueError): pass` внутри `tr()`. Добавлен плейсхолдер `{min_tps}` в обе локали.
* **[Фикс]** Иконка и логотип не находились вне собранного пакета: `_resolve_asset()` в `main_app.py` и захардкоженные пути в `jh_results_ui.py` искали только в директории `src/`. Файл `icon.ico` находится в корне проекта, `logo.png` — в `assets/`. Оба места теперь входят в цепочку поиска: `src/` → корень проекта → `assets/` → директория .exe → `_MEIPASS`.
* **[Чистка]** Удалён неиспользуемый импорт `ImageTk` из `main_app.py` (`from PIL import Image, ImageTk` → `from PIL import Image`).

</details>

<details>
<summary><b>📦 v2.0.2 — Многопоточность и надёжность</b></summary>

* **[Storage]** Полная переработка модели многопоточности. Два независимых, никогда не вложенных лока: `_file_lock` (только файловый I/O) и `_url_lock` (только мутации in-memory сетов). Всегда актуальные `_approved_urls` / `_rejected_urls`, заполняемые при старте — O(1) дедуп, без обращений к диску.
* **[Storage]** Удалено поле `description` (сырой `document.body.innerText`, до нескольких МБ) из одобренных записей. Раздувшийся файл удерживал `_file_lock` дольше 8 секунд, вызывая таймауты всех webhook-запросов.
* **[Storage]** `_migrate_strip_description()`: однократная стартовая миграция удаляет устаревшее поле из всех существующих записей до запуска сервера.
* **[Enqueue]** Удалены `_enqueue_lock` и `_in_flight_urls`. Тройная проверка дублей (сериализовавшая 24 Flask-потока) заменена одной O(1)-проверкой + безусловным `queue.put()`. Воркер выполняет лёгкий дедуп как страховку.
* **[Startup]** Исправлена гонка запуска Flask через `threading.Event` (`_flask_ready`): кнопка показывает «ЗАПУСК…» до привязки порта 5000.
* **[Extension]** Убрана проверка `tab.status === 'loading'` — SPA (LinkedIn, HH.ru) никогда не дают `'complete'`. Текст усекается до 50 000 символов — предотвращает зависание GIL при 24 параллельных воркерах.

</details>
